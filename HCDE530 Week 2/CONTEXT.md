# Project Context (HCDE 530 Week 2)

## Purpose

This project demonstrates how to process a data file effectively and clearly show the most important parts of the work.

## Primary Audience

Instructors are the primary audience.

## Top Priority

Output correctness is the highest priority.

## Constraints

- Keep everything beginner-friendly.
- Do not change file names or folder structure.
- Use plain language with no jargon.

## Data Expectations

This project should handle slightly messy real-world CSV data.

## Error Handling Preference

Fail fast with a short error message plus a fix hint.

Example style:
- `Missing required column: response. Check your CSV header names.`

## Script Run Experience

Either run location should work:
- From `HCDE530 Week 2`: `python3 demo_word_count.py`
- From repo root: `python3 "HCDE530 Week 2/demo_word_count.py"`

## Required Output Focus

For analysis output, prioritize:
- Top 5 longest responses by word count
- Show only word count and response text

## Code Communication Style

Use lightweight support across all of the following:
- Brief comments in the Python file
- Short written explanation file(s)
- Clear printed terminal output

## Dashboard Scope

`dashboard.html` is a supporting demo artifact, not the required source of truth.

## Out of Scope

- Advanced software-engineering patterns
- Jargon-heavy explanations
- Any file/folder structure changes
