# Week 2 — C2: Code literacy and documentation (resubmission)

This folder’s scripts read survey answers from a CSV and **count words** in each person’s response, then print a short summary (shortest, longest, average) so you can see how much people wrote in plain language. `demo_word_count.py` walks `demo_responses.csv` row by row with a readable table; `response_length_summary.py` does the same job in a reusable way—CSV beside the script via `Path(__file__)`, checks for a `response` column before looping so errors say what to fix, and accepts an optional filename on the command line. For C2 I added **docstrings** on `count_words` and `load_response_texts` that state inputs, outputs, and failure modes, plus **comments that explain why** (stable paths, argv for swapping data, newline handling in the demo). **This file** is the non-technical map of what runs where; **git** messages should name the behavior and reason (for example the Week 2 resubmission commit describes the documentation pass), not vague words like “update.”
